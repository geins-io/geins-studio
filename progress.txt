## Codebase Patterns

### Vue Component Structure
- Pages use `<script setup lang="ts">` with TypeScript
- Keep-alive is used for tab content to preserve state
- Tabs are managed via `currentTab` reactive state (0-indexed)
- Content structure: `ContentEditMainContent > ContentEditCard` for tab panels

### Data Display Patterns
- Use `ContentDataList` component for key-value pair displays
- DataItem interface: `{ label: string, value: string, displayType?: DataItemDisplayType }`
- Empty states should show placeholder text with `text-muted-foreground text-sm` classes

### Stores and State
- Access stores via `useXStore()` composables
- Use `storeToRefs()` to get reactive refs from stores
- `accountStore.currentLanguage` provides current UI language (defaults to 'en')
- Product localizations: `Record<string, Localizations>` where Localizations has: name, slug, text1, text2, text3

### Testing
- Tests use vitest and @nuxt/test-utils
- Test files go in `__tests__/` directories
- Use proper TypeScript interfaces for mock data with Record<string, T> for dynamic keys
- Computed properties should be tested for: full data, partial data, empty data, language switching
- Form validation should test: schema validation, submission logic, error handling, state management

### Type System
- Localized data uses `Localized<T>` = `{ [lang: string]: T }`
- Always provide proper type annotations for objects with string index signatures
- Product type includes optional `localizations?: Localized<Localizations>`

### Form Validation and Entity Operations
- Use zod for form validation schemas
- `useEntityEdit` composable provides `createEntity` and `updateEntity` methods
- Toast variants: 'positive' (success), 'negative' (error), 'warning', 'default'
- `createEntity` handles: validation, API call, success toast, navigation to detail page, error toast
- Form validation should be checked before submission with `form.validate()`
- Loading states are automatically managed by `useEntityEdit`
