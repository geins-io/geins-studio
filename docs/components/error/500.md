# `500`

`500` is the standard server-error page — used as the body of `error.vue` when Nuxt catches an unhandled error. Renders an `Empty` primitive with "Refresh" and "Home" actions and an optional collapsed error-details disclosure.

## Features

- Empty primitive with `ServerCrash` icon and localized title/description (`error.500_title`, `error.500_description`)
- "Refresh" calls `window.location.reload()` (full reload, drops in-memory state)
- "Home" emits `clear` so the parent can navigate
- Optional `<details>` block with the raw `errorDetails` — collapsed by default, gated by `showErrorDetails`

## Usage

```vue
<script setup lang="ts">
const props = defineProps<{ error: NuxtError }>();
const handleClear = () => clearError({ redirect: '/' });
const isDev = import.meta.dev;
</script>

<template>
  <Error500
    :message="error.message"
    :error-details="error.stack"
    :show-error-details="isDev"
    @clear="handleClear"
  />
</template>
```

## Props

### `message`

```ts
message?: string
```

Override the default localized description.

- **Default:** `''` (uses `$t('error.500_description')`)

### `errorDetails`

```ts
errorDetails?: string
```

Raw error text shown inside the `<details>` disclosure. Only rendered when `showErrorDetails` is `true`.

### `showErrorDetails`

```ts
showErrorDetails?: boolean
```

Reveals the `<details>` block. Gate this on `import.meta.dev` so production users never see stack traces.

- **Default:** `false`

## Events

### `clear`

Emitted when the user clicks Home. The parent should call `clearError` (or equivalent) to navigate.

## Dependencies

- shadcn-vue `Empty`, `EmptyMedia`, `EmptyHeader`, `EmptyTitle`, `EmptyDescription`, `EmptyContent`, `Button`
