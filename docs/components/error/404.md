# `404`

`404` is the standard not-found error page — used as the body of `error.vue` when Nuxt catches a 404. Renders an `Empty` primitive with "Go back" and "Home" actions.

## Features

- Empty primitive with `FileQuestion` icon and localized title/description (`error.404_title`, `error.404_description`)
- "Go back" uses `window.history.back()` when there's history, otherwise emits `clear` so the parent can navigate home
- "Home" emits `clear` — the parent decides where home goes

## Usage

Mount as the visual body of a Nuxt error page:

```vue
<script setup lang="ts">
const props = defineProps<{ error: NuxtError }>();
const handleClear = () => clearError({ redirect: '/' });
</script>

<template>
  <Error404 v-if="error?.statusCode === 404" :message="error.message" @clear="handleClear" />
  <Error500 v-else :message="error.message" @clear="handleClear" />
</template>
```

## Props

### `message`

```ts
message?: string
```

Override the default localized description.

- **Default:** `''` (uses `$t('error.404_description')`)

## Events

### `clear`

Emitted when the user clicks Home, or clicks Go back without browser history. The parent should call `clearError` (or equivalent) to navigate.

## Dependencies

- shadcn-vue `Empty`, `EmptyMedia`, `EmptyHeader`, `EmptyTitle`, `EmptyDescription`, `EmptyContent`, `Button`
