# shadcn-vue Primitives

Geins Studio uses [shadcn-vue](https://www.shadcn-vue.com) primitives as its base UI layer. Source lives in `app/components/ui/` and is initially generated by the shadcn-vue CLI. Because every primitive already has authoritative upstream documentation, we don't repeat it here. This page lists which primitives we consume and where in the codebase they appear, so you can jump from a Geins component to the upstream docs in one click.

> [!NOTE]
> Some primitives have been extended in-place — additional variants, props, or states added on top of the upstream version to fit Geins needs. Always read the actual source under `app/components/ui/<name>/` before assuming behaviour matches the upstream docs verbatim.

## Installing a new primitive

Always start with the CLI — manual files miss the registry hook-ups and break future upgrades. Local edits on top of the generated file are fine.

```sh
npx shadcn-vue@latest add <component>
```

## Notes

- Most shadcn-vue primitives are **auto-imported by Nuxt** — you don't need explicit imports for `Button`, `Badge`, `Tabs`, etc. inside `<template>`. The exception is `useToast`, which is imported explicitly: `import { useToast } from '@/components/ui/toast/use-toast'`.
- When a primitive needs a Geins-specific wrapper (theming, alias map, additional behavior), we build a thin component on top of it — see [ButtonIcon](/components/button/ButtonIcon) for the canonical pattern.
- Reka UI gotcha: `TabsContent` panels containing `<FormField>` components must use `force-mount class="data-[state=inactive]:hidden"` — switching tabs unmounts otherwise and VeeValidate clears values silently.
