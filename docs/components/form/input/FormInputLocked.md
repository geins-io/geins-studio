# `FormInputLocked`

`FormInputLocked` is the read-only display variant for a value the user can't edit — a styled `InputGroup` with a lock icon and an optional explainer tooltip.

## Features

- Disabled-looking but tab-skipped (`tabindex="-1"`, `cursor-not-allowed`)
- Lock icon in the trailing addon
- Optional tooltip on the lock icon for the "why" (e.g. "Set by Geins, contact support")
- Configurable `type` and `placeholder` like a normal text input

## Usage

### Basic Usage

```vue
<template>
  <FormInputLocked
    :model-value="account.publicId"
    tooltip="Public ID is generated by Geins and cannot be changed."
  />
</template>
```

## Props

### `modelValue`

```ts
modelValue?: string | null
```

The displayed value. Read-only — no `update:modelValue` is emitted.

### `type`

```ts
type?: string
```

Native input `type`. Use `'password'` to mask.

- **Default:** `'text'`

### `placeholder`

```ts
placeholder?: string
```

Placeholder when `modelValue` is empty.

### `tooltip`

```ts
tooltip?: string
```

When set, hovering the lock icon shows this tooltip.

## Dependencies
