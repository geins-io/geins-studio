## Context

The General tab is the administrative home for a channel's core identity fields. The channel edit page scaffold (M1/004) built the tab container with a placeholder body — this issue fills in the first card of the General tab with the three core identity fields from the Figma design (node `43:19846`).

## Expected Outcome

The General tab (`currentTab === 'General'`) renders a `ContentEditCard` titled "Details" containing three form fields wired to the form schema already established in the scaffold:
- **Display name** — customer-facing name (`displayName`)
- **Name** — internal/admin-only identifier, auto-generated from display name on creation (`name`), with helper text
- **URL** — the channel's storefront URL (`url`), with a warning helper text about link breakage

## Implementation Notes

- Use `ContentEditCard` with a "Details" title.
- Layout: `FormGrid design="1+1"` for Display name and Name side by side; full-width `FormGrid design="1"` for URL below.
- Use `FormField v-slot="{ componentField }"` pattern for all three inputs (already standard across the codebase).
- Helper text under Name: `"Generated by initial display name upon creation"` — use a `<p class="text-muted-foreground text-xs mt-1">` element below the input.
- Helper text under URL: `"Changing the URL will break existing links"` — same pattern; use a warning colour (e.g. `text-amber-600`) or add a Lucide `TriangleAlert` icon inline.
- In create mode, the Name field can be left blank (auto-generated by the backend) or editable — follow whatever the Zod schema in the scaffold requires (`z.string().min(1)` implies required; if the backend auto-generates it, relax to `z.string().optional()` in create mode only).
- All i18n keys must be added to both `i18n/locales/en.json` and `sv.json` under the `channel` namespace (e.g. `channel.display_name`, `channel.name`, `channel.url`, `channel.name_helper`, `channel.url_warning`).
- The card should only appear when `currentTab === 'General'`.

## Acceptance Criteria

- General tab shows the Details card with Display name, Name, and URL fields.
- `FormGrid design="1+1"` places Display name and Name side by side; URL is full-width.
- Helper text appears below Name and URL fields.
- All three fields are two-way bound to the form (changing them marks `hasUnsavedChanges` as true).
- `pnpm typecheck` and `pnpm lint:check` pass with no new errors.
- i18n: all visible strings are in `en.json` and `sv.json`; no hardcoded English in the template.
- Vitest: test that the card renders in edit mode with pre-filled values; test that URL helper text is present.
